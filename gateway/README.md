# بوابة واتساب بربط QR — سهم سكرينر

تربط **حسابك أنت** في واتساب (مسح QR مرة واحدة، كأنه واتساب ويب)، ثم تُرسل
رسالة إشارات اليوم — بالأهداف والأوقاف وتعليمات التنفيذ — إلى واتسابك
تلقائياً كل يوم في الوقت الذي تحدده.

> لماذا لا تعمل من Vercel مباشرة؟ ربط QR يحتاج عملية تبقى حية تحفظ جلسة
> واتساب، واستضافة Vercel لحظية بلا تخزين دائم. لذلك تعمل البوابة على أي
> جهاز عندك يبقى شغالاً (جهاز منزلي/مكتبي/سيرفر صغير)، وهي التي **تسحب**
> الرسالة الجاهزة من موقعك ثم ترسلها — لا تحتاج فتح منافذ ولا إعدادات شبكة.

## التشغيل على ويندوز (مرة واحدة)

1. ثبّت Node.js إن لم يكن عندك: https://nodejs.org (نسخة LTS)
2. انسخ مجلد `gateway` إلى الجهاز، ثم شغّل `start.bat`
3. أول مرة: سيُفتح ملف `.env` — ضع فيه قيمة `CRON_SECRET` (سر مسار الملخص
   في Vercel) واحفظ، ثم شغّل `start.bat` من جديد
4. سيظهر رمز QR في النافذة وفي المتصفح على `http://localhost:8899` —
   من جوالك: **واتساب ← الإعدادات ← الأجهزة المرتبطة ← ربط جهاز** وامسحه
5. سترى «✅ متصل» — اضغط زر «أرسل رسالة الإشارات الآن» للتجربة

من الآن فصاعداً: ما دامت النافذة تعمل، تصلك الرسالة يومياً في الوقت
المحدد في `.env` (الافتراضي 22:30 بتوقيت جهازك، أيام عمل السوق فقط).

## على لينكس/ماك

```bash
cd gateway
cp .env.example .env   # ثم ضع CRON_SECRET
npm install --omit=dev
npm start              # أو: npm run send-now للتجربة الفورية
```

## الإعدادات (`.env`)

| المتغير | المعنى | الافتراضي |
|---|---|---|
| `SITE_URL` | رابط منصتك | رابط النشر الحالي |
| `CRON_SECRET` | سر مسار `/api/digest` (نفسه في Vercel) | — |
| `SEND_TIME` | وقت الإرسال اليومي بتوقيت الجهاز | 22:30 |
| `SEND_DAYS` | أيام الإرسال (1=اثنين … 5=جمعة) | 1,2,3,4,5 |
| `SEND_TO` | `self` = لنفسك، أو رقم دولي بلا + | self |

## ملاحظات مهمة

- الجلسة تُحفظ في مجلد `auth/` — لا تشاركه مع أحد (يعادل جلسة واتساب ويب).
- إن سجّلت خروج الجهاز من واتساب، احذف مجلد `auth/` وأعد المسح.
- هذه طريقة غير رسمية (مكتبة Baileys مفتوحة المصدر تحاكي واتساب ويب) —
  للاستخدام الشخصي بمعدل رسالة يومية المخاطرة منخفضة، لكن Meta قد تقيد
  الأتمتة نظرياً. البديل الرسمي المدعوم دائماً: CallMeBot أو Cloud API.
- لإبقائها تعمل بعد إغلاق النافذة على ويندوز: شغّلها كمهمة مجدولة عند بدء
  التشغيل (Task Scheduler ← Create Task ← Run whether user is logged on or not).
