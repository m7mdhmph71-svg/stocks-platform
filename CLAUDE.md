# سهم سكرينر — Stocks Platform

منصة عربية (RTL بالكامل) لفرز الأسهم الأمريكية: فحص شرعي بمنهجية أيوفي 21
مع نسبة التطهير، ثلاثة فلاتر جاهزة بدلالات Finviz حرفية + فلتر مخصص،
أهداف/أوقاف فنية لكل استراتيجية، اختبار تاريخي يحاكي خطة الهدف/الوقف،
وحسابات مستخدمين (قائمة متابعة + سجل صفقات). المالك يخطط لتحويلها لمنتج
مدفوع برسوم رمزية على خادم خاص.

## الأوامر

```bash
npm run dev          # تطوير (بلا حسابات ما لم تضبط DATABASE_URL)
npm run build        # بناء الإنتاج — يجب أن يمر قبل أي دفع
npm run typecheck    # tsc --noEmit — صفر أخطاء دائماً

# التجربة المحلية الكاملة (حسابات + PostgreSQL) — على جهاز المستخدم:
docker compose -f docker-compose.local.yml up -d --build   # ثم localhost:3000
# أو نقرة مزدوجة: start-local.command (ماك) / start-local.bat (ويندوز)

# قاعدة البيانات
npx prisma db push   # مزامنة المخطط (يتطلب DATABASE_URL)
npx prisma generate  # يعمل تلقائياً postinstall
```

## البنية

- `src/lib/types.ts` — العقد المركزي لكل الأنواع. عدّله بحذر شديد.
- `src/lib/yahoo/` — مصدر البيانات: client.ts (كوكي+crumb مع إعادة محاولة
  429/5xx)، screener.ts (فرز خشن خادمي بترقيم صفحات)، quote.ts
  (quoteSummary للأساسيات، كاش 6 ساعات)، chart.ts (شموع يومية).
- `src/lib/shariah/` — الفحص الشرعي: نشاط محرم (قوائم صناعات + أنماط نصية
  في وصف الشركة) ثم نسب 30/30/5، ونسبة التطهير (مسقوفة 100%).
- `src/lib/targets/` — المؤشرات (Wilder RSI/ATR، SMA، بيفوت، قيعان/قمم
  متأرجحة fractal) ومحرك الأهداف/الوقف. صيغتان للمضاربة: «الهيكلية»
  (وقف تحت آخر قاع متأرجح + أهداف على مستويات فعلية بحارس عائد/مخاطرة
  ≥ 1) و«الكلاسيكية» (ATR والمحاور). الافتراضي **لكل استراتيجية** قرار
  تجريبي في defaultFormulaFor: الزخم → كلاسيكية، السيولة → هيكلية —
  أعد تدقيقه دورياً بزر المقارنة قبل تغييره.
- `src/lib/filters/` — presets.ts فيه فلاتر المستخدم الثلاثة **بحروفها** —
  لا تغيّر دلالاتها أبداً؛ engine.ts تقييم الشروط؛ longterm.ts بوابتا
  الاستثمار الأساسية والفنية؛ saudi.ts تبويب السوق السعودي (تداول،
  region=sa في فرز ياهو، أسعار بالريال عبر currencyFor).
- `src/lib/backtest/` — محاكاة تاريخية: دخول بإغلاق الإشارة، خروج عند
  الهدف الأول أو الوقف (الوقف يُغلَّب عند التلامس المزدوج، وفجوات الافتتاح
  تُحتسب بسعر الافتتاح)، أفق 15 جلسة. ومقارنة 6 صيغ خروج عبر ?compare=1
  (كلاسيكية/هيكلية/متحركة/هجينة + توقيت خالص: ليلة/جلسة واحدة).
  النتيجة التجريبية (يوليو 2026، 30 جلسة): الزخم → الكلاسيكية أفضل
  (معامل 1.20، إصابة 74%)، السيولة → الهيكلية (1.06)؛ صيغ التتبع
  خاسرة بوضوح على كون 1-10$ (ينعكس سريعاً)، ولا حافة في «ليلة واحدة»
  (لا فجوات صباحية منتظمة). العينات صغيرة — أعد التدقيق مع اتساع النافذة.
- `src/app/api/` — screener، stock/[ticker]، backtest، quotes، search،
  candles (شموع بكل الفترات لصفحة السهم)، digest (ملخص واتساب اليومي)،
  alerts (محرك التنبيهات اللحظية — تسحبه البوابة، منع التكرار عبر SentAlert)،
  whatsapp/{gateway,prefs,command} (وسيط ربط QR من المنصة، تفضيلات التنبيهات،
  الأوامر التفاعلية)، auth/* (دخول/تسجيل بحد محاولات في الذاكرة +
  forgot/reset برمز 6 أرقام عبر البريد أو واتساب + verify لتوثيق البريد
  — البريد عبر Resend بلا تبعيات: src/lib/mail.ts)، watchlist، trades،
  purification،
  admin/db-init (تهيئة الجداول للنشرات بلا خدمة ترحيل، محمي بـ CRON_SECRET،
  قابل لإعادة التشغيل — يتخطى الموجود وينفّذ UPGRADE_SQL دائماً)،
  admin/set-plan (تفعيل الاشتراك يدوياً قبل بوابة الدفع، بنفس السر).
- `src/lib/plan.ts` — حدود FREE/PRO المصدر الوحيد (PLAN_LIMITS): المجاني
  فرز + فحص شرعي + تطهير (التطهير عبادة لا تُباع) بقوائم 10 ونافذة اختبار
  10 جلسات؛ الاحترافي يضيف التنبيهات وأوامر الحساب في واتساب والمقارنة
  ونوافذ كاملة. الخطة الفعلية عبر effectivePlan (تحترم planExpires)،
  وصفحة /pricing تُولَّد من PLAN_LIMITS نفسها.
- `prisma/schema.prisma` — User (خطة FREE/PRO + emailVerified)، WatchItem
  (يرصد تغيّر الحكم الشرعي + alertVerdict خط أساس التنبيهات)، Trade (سجل
  صفقات بهدف/وقف)، NotifyPref (تفضيلات التنبيهات لكل مستخدم)، SentAlert
  (مفتاح فريد يمنع تكرار التنبيه)، AuthToken (رموز استرجاع/توثيق مهشّرة
  بمحاولات وصلاحية). ملاحظة: تغيير كلمة المرور لا يبطل جلسات JWT القائمة
  (30 يوماً) — يُعالج عند مرحلة الدفع بإصدار جلسات قابلة للإبطال.
- `gateway/` — بوابة واتساب اختيارية بربط QR (Baileys) تعمل على جهاز
  المستخدم: ملخص يومي مجدول، سحب /api/alerts دورياً، رد على الأوامر
  الواردة عبر /api/whatsapp/command، وواجهة JSON محمية بالسر تخدم
  صفحة «تنبيهاتي» في المنصة (GATEWAY_URL).

## قواعد غير قابلة للتفاوض

- كل نص واجهة بالعربية الفصحى، أرقام غربية (0-9)، `dir="ltr"` على الرموز
  اللاتينية داخل نص عربي.
- لا تلمس صياغات إخلاء المسؤولية (ليست توصية / الفحص الشرعي تقديري) —
  موجودة في التذييل وصفحة السهم و/terms.
- TypeScript strict: صفر أخطاء tsc قبل أي commit.
- بلا مكتبات رسم — SVG يدوي فقط. بلا تبعيات جديدة دون سبب قوي.
- طبقة البيانات تتدهور بأمان: Yahoo → (Finviz Elite إن وجد توكن) → demo
  مع شريط تنبيه. لا تكسر هذا التسلسل.
- الحسابات اختيارية: بلا DATABASE_URL كل ميزات الحسابات تختفي بسلاسة
  ولا شيء ينكسر (dbEnabled() في src/lib/db.ts).

## متغيرات البيئة

انظر `.env.example` (المنصة) و`deploy/.env.server.example` (الخادم الخاص).
المفاتيح: DATABASE_URL، AUTH_SECRET، CRON_SECRET، COOKIE_SECURE=0 للتجربة
المحلية عبر http (سفاري يرفض كوكي Secure على localhost)، ومزوّدا واتساب
الاختياريان (CALLMEBOT_* أو WHATSAPP_*).
